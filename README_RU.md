# ROPDS

[![CI](https://github.com/dshein-alt/ropds/actions/workflows/tag-tests.yml/badge.svg?branch=master)](https://github.com/dshein-alt/ropds/actions/workflows/tag-tests.yml)

Быстрый и лёгкий сервер персональной электронной библиотеки с каталогом OPDS 1.2 / 2.0 и веб-интерфейсом. Написан на Rust.

> Создан по мотивам [SimpleOPDS](https://github.com/mitshel/sopds), переписан с нуля как современное решение для домашнего сервера или небольшого VPS.

[English version](README.md)

## Что внутри

- Один исполняемый файл на Rust, никаких внешних зависимостей, контейнеры по желанию
- Каталог OPDS 1.2 / 2.0, совместимый с популярными программами для чтения элекронных книг (CoolReader, FBReader, Librera, KOReader и др.)
- Веб-интерфейс: каталог, поиск, админка, загрузка книг, встроенная читалка
- Многопользовательский режим с индивидуальными правами на загрузку
- Вход через OAuth (Google, Yandex, Keycloak OIDC) с очередью на одобрение и уведомлениями на почту
- Фоновое сканирование библиотеки, включая ZIP-архивы и файлы INPX
- Светлая и тёмная тема, можно установить как PWA

Идея проста: сервер библиотеки, который достаточно настроить один раз. Разворачивается за пару минут — для себя, семьи и друзей.

Проект одновременно служит **учебной площадкой** для освоения экосистемы Rust и экспериментов с **LLM-агентами для написания кода**.

## Быстрый старт

### 1. Сборка

```bash
cargo build --release
```

### 2. Настройка

```bash
cp config.toml.example config.toml
```

Минимально необходимые параметры:

```toml
[server]
base_url = "http://localhost:8081"

[library]
root_path = "/path/to/books"
```

Полный пример с обложками и базой данных:

```toml
[server]
base_url = "http://localhost:8081"

[library]
root_path = "/path/to/books"

[covers]
covers_path = "/path/to/books/covers"
cover_max_dimension_px = 600
cover_jpeg_quality = 85
show_covers = true

[database]
url = "sqlite://ropds.db?mode=rwc"
```

### 3. Создание администратора

```bash
./target/release/ropds --set-admin <пароль>
```

### 4. Запуск

```bash
./target/release/ropds
```

Откройте:

- Веб-интерфейс: `http://localhost:8081/web`
- OPDS: `http://localhost:8081/opds`

### Однократное сканирование

Просканировать библиотеку один раз без запуска сервера:

```bash
./target/release/ropds --scan
```

## Возможности

### Управление библиотекой

- Фоновое сканирование по расписанию (cron-формат)
- Параллельное сканирование с динамическим распределением задач и ограничением числа потоков
- Прозрачная работа с книгами внутри ZIP-архивов и с индексами INPX
- Извлечение метаданных из FB2, EPUB и MOBI — название, авторы, жанры, серии, обложки, аннотации
- Генерация обложек для PDF и DjVu через внешние утилиты (`pdftoppm`, `ddjvu`)

### Каталог OPDS

- Полноценные фиды OPDS 1.2 / 2.0 с постраничной навигацией
- Просмотр по авторам, сериям, жанрам, каталогам и алфавитному указателю
- Поддержка OpenSearch
- Миниатюры и полноразмерные обложки
- HTTP Basic Auth (при необходимости отключается)

### Поиск

- Полнотекстовый поиск по названиям, авторам и сериям — и в OPDS, и в веб-интерфейсе
- Алфавитная навигация с настраиваемым порогом разбиения для больших коллекций
- Дескриптор OpenSearch для интеграции с OPDS-клиентами

### Книжная полка

- У каждого пользователя свой список чтения — добавление и удаление в один клик
- Книги попадают на полку автоматически при скачивании
- Сортировка по дате добавления, названию или автору, по возрастанию или убыванию
- Бесконечная прокрутка

### Загрузка книг

- Загрузка прямо через веб-интерфейс (FB2, EPUB, PDF и другие форматы)
- Метаданные разбираются автоматически — название, авторы и жанры можно подправить перед сохранением
- Право на загрузку настраивается администратором для каждого пользователя

### Жанры

- Иерархическая система: разделы и подкатегории
- Названия жанров с переводами на разные языки, хранятся в базе данных
- Управление через админку: создание разделов, добавление жанров, редактирование переводов
- Гибкая привязка — у книги может быть несколько жанров, менять их можно в любой момент

### Управление пользователями

- Многопользовательский режим со встроенной панелью администрирования
- Создание и удаление пользователей, сброс паролей, управление правами на загрузку
- Пользователь сам меняет отображаемое имя и пароль в профиле
- OAuth-пользователи могут сгенерировать отдельный пароль для OPDS
- Принудительная смена пароля при первом входе, если это задал администратор

### OAuth и заявки на доступ

- Провайдеры: Google, Yandex, Keycloak (OIDC)
- Новые OAuth-пользователи попадают в очередь ожидания, пока администратор не подтвердит доступ
- Администратор может одобрить, отклонить, заблокировать или восстановить заявку
- При одобрении можно привязать OAuth-аккаунт к уже существующему локальному пользователю
- Для Keycloak доступно автоматическое одобрение и маппинг ролей в права загрузки и администрирования
- По желанию — уведомления администратору на почту (SMTP) о новых и повторных заявках

### Встроенная читалка

- EPUB, FB2, MOBI, DjVu и PDF открываются прямо в браузере — ничего скачивать не нужно
- Позиция чтения сохраняется и восстанавливается автоматически для каждого пользователя и книги
- Боковая панель с историей чтения и быстрым переходом к недавним книгам
- В обычном браузере открывается в новой вкладке, в установленном PWA — в том же окне
- Работает на базе [foliate-js](https://github.com/johnfactotum/foliate-js) и [djvu.js](https://github.com/RussCoder/djvujs)

### Веб-интерфейс

- Адаптивная вёрстка на Bootstrap 5, светлая и тёмная тема
- Можно установить как PWA на телефон или компьютер (manifest + service worker)
- Навигация по каталогам, авторам, сериям и жанрам с хлебными крошками
- Редактирование метаданных книги прямо на странице (для администраторов)
- Страница дубликатов: группировка одинаковых изданий по названию и авторам, с пагинацией
- Предпросмотр обложки, полноразмерный показ по клику

### Локализация

- В комплекте **английская** и **русская** локали
- Файлы локалей — обычный TOML; чтобы добавить язык, достаточно скопировать `en.toml` и перевести строки
- Названия жанров хранятся в базе данных с переводами на разные языки
- Язык интерфейса запоминается для каждого пользователя в cookie

### Безопасность

- Хэширование паролей Argon2
- Подпись сессионных кук HMAC-SHA256
- Настраиваемое время жизни сессии
- Индивидуальные права на загрузку книг
- Роль суперпользователя для доступа к администрированию

## Настройка

Все параметры — в файле `config.toml`. Полный пример с комментариями: [config.toml.example](config.toml.example).

Параметр `server.base_url` обязателен — он используется для формирования OAuth callback URL и ссылок в уведомлениях администратору.

| Секция | Что настраивается |
|---|---|
| `[server]` | Адрес, порт, уровень логирования, секрет сессии, TTL, `base_url` |
| `[library]` | Путь к книгам, расширения файлов, поддержка ZIP/INPX |
| `[covers]` | `covers_path`, размер и сжатие обложек (`cover_max_dimension_px`, `cover_jpeg_quality`), `show_covers` |
| `[database]` | URL подключения — `sqlite://`, `postgres://` или `mysql://` |
| `[opds]` | Название каталога, пагинация, авторизация |
| `[scanner]` | Расписание (cron), число потоков, проверки целостности |
| `[web]` | Язык по умолчанию (`en`, `ru`), тема (`light`, `dark`) |
| `[upload]` | Загрузка книг: вкл/выкл, директория, лимит размера |
| `[reader]` | Встроенная читалка: вкл/выкл, размер истории чтения |
| `[oauth]` | Провайдеры, модерация, маппинг ролей Keycloak, уведомления |
| `[smtp]` | Настройки SMTP для исходящих уведомлений |

## Вход через OAuth и одобрение доступа

1. Укажите в `server.base_url` внешний адрес сервера.
2. Настройте хотя бы одного провайдера в `[oauth]` (`google_*`, `yandex_*` или Keycloak).
3. По желанию включите уведомления: `oauth.notify_admin_email = true` и заполните `[smtp]`.
4. Пользователь входит через `/web/login`.
5. Новые пользователи попадают в раздел **Админка -> Заявки на доступ** и ждут одобрения.

Минимальный пример (Google + уведомления администратору):

```toml
[server]
base_url = "https://books.example.com"

[oauth]
google_client_id = "..."
google_client_secret = "..."
notify_admin_email = true

[smtp]
host = "smtp.example.com"
port = 587
username = "smtp-user"
password = "smtp-pass"
from = "ropds@example.com"
send_to = ["admin@example.com", "alerts@example.com"]
starttls = true
```

## Развёртывание

### Systemd

Используйте шаблон юнита `service/ropds.unit` (сервис работает от пользователя `ropds`).

```bash
sudo useradd --system --home /opt/ropds --shell /usr/sbin/nologin ropds || true
sudo install -d -o ropds -g ropds /opt/ropds
sudo install -m 0755 target/release/ropds /opt/ropds/ropds
sudo install -m 0644 config.toml /opt/ropds/config.toml
sudo install -m 0644 service/ropds.unit /etc/systemd/system/ropds.service
sudo systemctl daemon-reload
sudo systemctl enable --now ropds.service
sudo systemctl status ropds.service
sudo journalctl -u ropds.service -f
```

## Поддерживаемые форматы

| Формат | Метаданные | Обложки |
|---|---|---|
| FB2 | Полные (название, авторы, жанры, серии, аннотация, язык) | Встроенные |
| EPUB | Полные (OPF) | Встроенные |
| MOBI | Полные (название, автор, описание, язык, дата) | Встроенные |
| PDF | Частично (название, автор через `pdfinfo`) | Первая страница (через `pdftoppm`) |
| DjVu | Только имя файла | Первая страница (через `ddjvu`) |

Книги внутри **ZIP-архивов** сканируются прозрачно. Файлы **INPX** поддерживаются как альтернатива сканированию отдельных архивов.

## База данных

SQLite используется по умолчанию и не требует никакой настройки. PostgreSQL и MySQL/MariaDB тоже поддерживаются — достаточно указать URL в `[database].url`.

**Поддерживаемые версии** (проверено в CI и end-to-end):

| Backend      | Минимум | Примечания                                                                          |
|--------------|---------|-------------------------------------------------------------------------------------|
| SQLite       | 3.35+   | Встроен через `sqlx`; ничего ставить не нужно.                                      |
| PostgreSQL   | 16+     | Проверено на 16 и 17. Более старые PG могут работать, но не тестируются.            |
| MariaDB      | 11+     | Проверено на 11.x и 12.x.                                                           |
| MySQL        | 8+      | Проверено на 8.4. Работает при штатном SQL-режиме `ONLY_FULL_GROUP_BY`.             |

При использовании SQLite обычно достаточно `workers_num = 2..4` — при большем числе потоков могут возникать блокировки записи во время крупных пересканирований.

Миграции применяются автоматически при запуске. Наборы миграций для каждого бэкенда встраиваются в исполняемый файл при сборке и выбираются по префиксу URL (`sqlite://`, `postgres://`, `mysql://`).

### Перенос данных между бэкендами (SQLite -> PostgreSQL или MySQL/MariaDB)

Выполняется в четыре шага:

1. **Создание роли и БД** на целевом сервере (один раз). Роли достаточно владеть БД — суперпользователь / `root` не нужны.
2. **Подготовка схемы через `ropds --init-db`** — создаёт БД (если её нет), применяет все миграции, очищает каждую пользовательскую таблицу, чтобы целевая БД была действительно пустой, и выходит. Откажется работать, если в БД уже есть строки (поэтому команду безопасно случайно запустить на живой или уже наполненной БД — в сообщении об ошибке будет указано, что нужно очистить вручную).
3. **Копирование данных скриптом `scripts/migrate_sqlite.py`** — минимальный помощник: truncate + copy + verify в одной CLI-сессии. Проверка: каждая таблица данных в целевой БД должна иметь 0 строк (это состояние после `--init-db`); иначе скрипт перечислит виновников и откажется работать. Требует интерактивного подтверждения; зависит только от `psql` или `mysql`/`mariadb`. Поддерживает запуск этих CLI внутри контейнера через `--db-container NAME --container-runtime {docker,podman}`.
4. **Запуск ROPDS** против нового URL.

- Инструкция на английском (с примерами SQL для PG и MySQL/MariaDB): [`scripts/README.md`](scripts/README.md)
- Инструкция на русском: [`scripts/README_RU.md`](scripts/README_RU.md)
